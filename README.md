# Telegram-агрегатор подій Києва “Куди піти Київ” (kyiv-event-guide)

Повноцінний MVP сервісу автоматичного збору, AI-модерації та публікації подій столиці у Telegram-канал з адмін-панеллю та користувацьким ботом.

---

## 🚀 Основні функції MVP

1. **Користувацький Telegram-бот**:
   - Швидкі фільтри подій за кнопками: *Сьогодні*, *Завтра*, *Вихідні*, *Безкоштовно*, *Для побачення*, *З дітьми*, *Тусовки*.
   - Виводить до 7 актуальних подій з бази у гарному форматі із посиланнями на квитки та картинками.
   - Подача заявки організаторами за допомогою покрокового FSM-сценарію `/submit`.
2. **Адмін-панель (React + Tailwind + Dark Mode)**:
   - **Dashboard**: Загальна статистика черг, активних джерел, нових raw матеріалів та журнал критичних помилок парсерів.
   - **Sources CRUD**: Керування джерелами (додавання сайтів, RSS-стрічок та Telegram-каналів) + кнопка миттєвого ручного збору даних.
   - **Raw Items**: Перегляд сирого контенту та кнопка повторної обробки за допомогою ШІ.
   - **Events Moderation (Split Screen)**: Інтерфейс модерації подій. Зліва — форма редагування, справа — інтерактивне візуальне прев'ю майбутнього поста в Telegram. Підтримка відхилення, публікації, планування на час, склеювання дублів та перегенерації AI-тексту.
   - **Submissions**: Керування ручними заявками з бота. Прийняття автоматично створює подію та відкриває редактор.
   - **Posts**: Історія публікацій та ручне створення дайджестів.
3. **Модулі збору даних (Scrapers)**:
   - **RSS-парсер**: Автоматично зчитує стрічки новин (Vgorode, MoeMisto тощо).
   - **Website Crawler**: Регулярно сканує сайти (Kyivmaps, Concert.ua, Pokupon) за посиланнями та завантажує сторінки деталей.
   - **Telegram Channel Monitor**: Сканує публічні канали за допомогою `Telethon` (якщо є API ID/HASH) або через **надійний веб-fallback без авторизації (`t.me/s/{username}`)**.
4. **AI-сервіс та Дублікати**:
   - Інтеграція з OpenAI API (`gpt-4o-mini`) для автоматичного парсингу факту події, дати, цін, визначення категорії, оцінки якості та пошуку дублікатів.
   - **Mock-режим**: Якщо API-ключ відсутній, сервіс використовує вбудований regex/keyword парсер, що дозволяє повноцінно протестувати систему безкоштовно!
   - Автоматичне склеювання дублів у вікні ±3 днів за схожістю назв, локацій та дати.

## 🛠️ Технологічний стек

- **Backend**: Python 3.11, FastAPI, SQLAlchemy, Alembic, PostgreSQL, Redis, Celery, Celery Beat.
- **Telegram Bot**: aiogram 3.x, Telethon (для TG моніторингу).
- **Frontend / Admin**: React 18, TypeScript, Vite, Tailwind CSS.
- **AI**: OpenAI API / Fallback Regex.
- **Деплой**: Docker Compose.

## ⚙️ Покроковий запуск (Docker)

### 1. Підготовка Telegram
1. Створіть бота в Telegram через [@BotFather](https://t.me/BotFather), отримайте `TELEGRAM_BOT_TOKEN`.
2. Встановіть команди для бота через `/setcommands` у BotFather:
   ```text
   start - Головне меню та привітання
   today - Події на сьогодні
   tomorrow - Події на завтра
   weekend - Події на вихідні
   free - Безкоштовні події
   kids - Розваги з дітьми
   date - Ідеї для побачень
   party - Вечірки та тусовки
   submit - Запропонувати свою подію
   ```
3. Створіть публічний Telegram-канал та додайте туди вашого бота як **Адміністратора** з правом публікації повідомлень. Отримайте ID каналу (наприклад, `-1002447952932` або юзернейм `@my_channel`).

### 2. Налаштування середовища `.env`
У кореневій папці проекту створено файл `.env`. Заповніть ваші змінні:
```env
# База даних (залишити за замовчуванням для Docker)
DATABASE_URL=postgresql+asyncpg://postgres:postgres@postgres:5432/kyiv_events
REDIS_URL=redis://redis:6379/0

# Telegram
TELEGRAM_BOT_TOKEN=8953691774:AAHvpPum5PG_8VjoDoay_Z8gpGodhSjfdzc
TELEGRAM_CHANNEL_ID=-1002447952932

# ШІ (якщо openai - вкажіть ключ, інакше залиште 'mock' для тестування без API)
AI_PROVIDER=mock
OPENAI_API_KEY=

# Адмін
ADMIN_EMAIL=admin@kyivevents.com
ADMIN_PASSWORD=adminpassword123
SECRET_KEY=9a8d7c6b5e4f3a2b1c0d9e8f7a6b5c4d
```

### 3. Запуск сервісів
Виконайте наступну команду в терміналі в корені проекту:
```bash
docker compose up --build
```
Ця команда автоматично:
1. Запустить бази даних Postgres та Redis.
2. Проведе міграції Alembic (`alembic upgrade head`).
3. Запустить `db_seed.py` (створить адмін-користувача `admin@kyivevents.com`, базові категорії та демо-джерела).
4. Запустить FastAPI на `http://localhost:8000`.
5. Запустить React на `http://localhost:5173`.
6. Запустить фонового Celery-воркера, Beat-планувальник та Telegram-бота.

## 💻 Використання системи

### 1. Адмін-панель
Відкрийте браузер на сторінці `http://localhost:5173`.
1. Увійдіть під даними `admin@kyivevents.com` / `adminpassword123`.
2. Перейдіть до **Джерел даних**, увімкніть потрібне джерело (наприклад, `Vgorode` або `Kyivmaps`) та натисніть **Запустити**, щоб негайно запустити збір подій.
3. Перейдіть до розділу **Модерація подій**. Свіжозібрані події з'являться у списку *Потрібен огляд (Needs Review)*.
4. Відредагуйте картку (якщо необхідно), оцініть прев'ю, перевірте дублі та натисніть **Публікувати**.

### 2. Збір та AI-обробка
- Кожні 30 хвилин система перевіряє RSS та Telegram-канали, кожну годину — сайти.
- Отримані сирі матеріали перетворюються на події за допомогою AI. Події з низькою якістю (наприклад, без дати, локації чи опису) автоматично йдуть до *Чернеток (Draft)*. Події високої якості потрапляють до *Needs Review*.
- Застарілі події щоночі переносяться до *Архіву*.
- Щоденні дайджести публікуються автоматично о 08:00 (сьогодні) та о 18:00 (завтра).
